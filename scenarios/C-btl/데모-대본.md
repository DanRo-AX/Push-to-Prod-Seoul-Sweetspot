# octopus 시연 대본 — 달무드 BTL 제안 흐름

> 길이 ~2.5분 · 화면 녹화 + 보이스오버. UI 한국어. 시나리오 `C-btl`.
> 검증 수치 그대로 사용 — 비트5 반응 인구 **0% → 78%**.

## 0. 사전 준비 (녹화 전)

- [ ] `OCTOPUS_SCENARIO=C-btl` 로 dev 실행, 브라우저 Chrome/Edge.
- [ ] 데모용 작업 폴더 하나 준비(예: `~/Demo/달무드/`). 그 안에 RFP PDF + `데모-기획안.md` 복사해 둠.
- [ ] 이미 한 번 폴더 열어 RFP 추출까지 돌려 캐시 워밍(녹화 중 72초 대기 방지).
- [ ] 우측 군집 패널에서 드래그할 군집: **P1 · P3 · P4** (선택 토글 미리 안 누름 — 녹화에서 보여줌).
- [ ] 창 폭 충분히(좌 대화 · 중앙 보드 · 우 군집 3분할 보이게).

## 촬영 노트

- ★ 비트 4·5가 가치 증명. 여기 호흡 길게, 나머지 빠르게.
- 비트 4·5는 실제 LLM 호출(수 초~십 초). '검토 중' 스피너 보여주거나 컷 편집으로 메움.
- 비트 5 수치는 모델 호출이라 매번 정확히 78%는 아님(70%대 전후). "0%에서 70%대로" 로 멘트하면 안전.
- P2는 정직하게 무반응 유지, P5는 mute — "아첨 안 한다" 한 마디 얹으면 신뢰 ↑.
